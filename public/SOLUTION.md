# 智阅云产品雷达：输入—处理—输出完整方案

## 1. 产品目标

服务产品经理、客服和需求承接团队，把分散在应用市场、客服工单和销售登记中的 CSV 反馈转成可解释、可追溯、可执行的产品决策。当前版本只接入 CSV，不解析聊天记录和邮件 TXT。

成功标准：产品经理可以在5分钟内确认本期 Top 痛点、上升问题、正面卖点和下一步行动；客服能得到受知识库约束的回复草稿；产品团队能在需求入池前完成语义查重。

## 2. 输入层

### 2.1 输入文件

| 数据 | 核心字段 | 用途 |
|---|---|---|
| 应用商店评论 | 日期、平台、评分、评论、点赞 | 大规模用户声音、趋势与热度 |
| 问题反馈表 | 客户、问题、备注、紧急程度 | 高质量问题、客户影响和处理上下文 |
| 需求登记表 | 客户、需求、价值、期望时间 | 商业价值和候选产品需求 |
| 角色表 | 昵称、角色、客户等级 | VIP/普通用户及内部角色影响权重 |
| 需求池 | ID、标题、描述、状态、优先级 | 新需求语义查重与关联 |
| 标注子集 | 主题、情感 | 独立质量评估 |

### 2.2 统一反馈模型

不同表转换为统一记录：`id / date / channel / author / role / text / rating / likes / urgency / sourceType`。处理结果继续增加：`topic / sentiment / feedbackType / severity / signal / confidence`。

### 2.3 数据质量处理

- UTF-8 BOM、引号内逗号和多行字段由状态机 CSV 解析器处理。
- 问题反馈表中未转义的客服备注逗号，通过“前四列固定、最后一列紧急程度、中间合并”修复。
- 需求池中未转义的描述逗号，通过日期字段定位并重建描述、提出方、状态和优先级。
- 不以评分直接代替文本情感；评分只作为弱特征。
- 原文始终保留，任何聚类和行动都可以返回证据。

## 3. 处理层

### 3.1 文本规范化

- 英文统一小写、全角术语归一，如 API、Excel。
- 修正常见错字和变体，如“闪腿→闪退”“fa票→发票”。
- 规范部分表情标记，同时保留反讽线索。
- 业务实体词包括发票、票据、合同、表格、回单、PDF、Excel、API、小程序。

### 3.2 噪声过滤

强规则识别互赞互评、广告导流、代开发票、错下应用、打卡及无信息短评。极短文本只有在不包含业务实体和主题关键词时才判为噪声，避免误删“闪退”“429”等短而重要的信号。

### 3.3 智阅云定制主题词库

主题体系与人工标注保持一致，共16类：

- 识别质量：印章遮挡、表格导出、其他字段
- 稳定性：批量上传、闪退与登录
- 性能与兼容：性能、文件格式
- 开放能力：API与集成
- 产品体验：广告弹窗、其他体验
- 商业与服务：价格计费、客服服务、数据隐私
- 产品发展：新功能需求、整体评价、噪声无关

关键词不是简单等权匹配。印章、批量上传、表格错位等高风险领域词具有更高权重；出现“印章/盖章/价税合计”时，领域优先规则会阻止其被泛化为“其他字段错误”。“希望、建议、能不能”只描述需求意图，具体业务主题优先。

### 3.4 情感分析

采用可解释组合评分：正负词、程度词、业务风险词、转折结构、弱星级特征和反讽模式。

重点处理：

- “每天先玩关闭弹窗小游戏，五星”识别为负面；
- “一星是因为太好用了”识别为正面；
- “修复了、改回五星”识别为正面恢复反馈；
- “总体不错，但是印章遮挡是硬伤”以问题负面为主。

### 3.5 反馈类型

输出噪声、咨询、功能需求、问题/Bug、体验问题、表扬和一般反馈。类型与主题分开：例如“希望优化印章遮挡”意图像需求，但在产品决策中仍属于识别质量问题。

### 3.6 聚类和热点排序

当前冷启动版本采用“固定主题语义聚类”：同一业务本质的多种表述归入同一主题，稳定、可解释且可以直接用标注集评估。

优先分由以下维度加权归一：

`28%规模 + 22%负面强度 + 18%业务严重度 + 12%影响面 + 10%时间趋势 + 10%客户价值`

严重度额外考虑金额/财务风险、数据丢失、整批失败、返工、紧急、月底、续约、重复发生等业务词。VIP权重用于反映商业影响，但不会单独决定最终优先级。

### 3.7 需求池语义查重

从反馈和需求池提取领域概念，计算概念重叠并补充同义规则：银行回单≈银行流水、连续拍摄≈批量扫描、主动回调≈异步回调/Webhook、团队子账号≈成员用量。

- 高相似：关联已有需求；
- 中相似：人工复核；
- 低相似：生成候选新需求。

### 3.8 回复草稿

咨询和高热负面反馈按内置的智阅云知识事实生成草稿，覆盖额度、隐私、API 429、文件格式、批量上传和印章遮挡。草稿不自动发送，不编造上线日期；涉及财务风险、隐私和VIP续约必须人工确认。

## 4. 输出层

### 4.1 一页洞察总览

- 反馈总量、有效信号率、负面量、主题数、候选行动数；
- 高频主题规模和负面率；
- 中文票据识别、速度、字段准确性等正面卖点；
- Top 3痛点及规模、独立用户、负面率、建议行动。

当前数据得到的前三项为：印章遮挡金额错误、批量上传超时、合并单元格导出错位。它们与客服工单、VIP影响、研发结论和已有需求状态相符。

### 4.2 聚类证据页

展示每个主题的优先分、规模、用户数、覆盖渠道、负面率、趋势、状态、责任团队、三条高信号代表反馈、正式建议和临时方案。

### 4.3 行动中心

以三列看板区分高优问题/Bug、关联已有需求、候选新需求。卡片可以打开需求查重结果和回复草稿，模拟人工确认后的工作流闭环。

### 4.4 评估页

基于150条人工标注子集展示主题准确率、主题Macro-F1和情感Macro-F1。本地规则基线当前实测：主题准确率74.0%、主题Macro-F1约69%、情感准确率87.3%、情感Macro-F1约87%。

正式答辩应说明：当前指标用于工程基线验证；若继续调词库，需要冻结一部分未使用样本作为最终测试集，避免数据泄漏。

## 5. 后续升级

1. 使用中文句向量做开放式聚类，发现固定标签之外的新问题。
2. 将现有规则作为弱标签，训练监督主题和情感分类器。
3. 增加多意图拆分，使一行CSV中的多个事项变成独立反馈原子。
4. 增加按周趋势、主题爆发检测和版本发布前后对比。
5. 对接正式需求系统和客服系统，但继续保留人工确认门槛。
