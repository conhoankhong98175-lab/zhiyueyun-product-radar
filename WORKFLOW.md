# Workflow：反馈信号到产品行动的闭环

## 1. 流程目标

将本地CSV反馈稳定地转换为“有证据、有优先级、有责任人、有下一步”的行动对象，同时保证任何对外承诺和正式系统写入均由人确认。

## 2. 参与角色

| 角色 | 主要职责 |
|---|---|
| 产品经理 | 确认问题定义、排序和需求去向 |
| 客服 | 补充上下文、审核回复草稿、联系用户 |
| 销售/客户成功 | 确认客户价值、续约和签约风险 |
| 研发/算法 | 接收Bug、确认技术状态和临时方案 |
| AI分诊引擎 | 清洗、分类、聚类、排序、查重和生成草稿 |

## 3. 标准工作流

### Step 1：准备CSV

负责人将应用市场评论、问题反馈表、需求登记表、角色表、需求池和标注集导出为UTF-8 CSV。每份文件保留原始ID和时间字段，避免手工合并导致证据丢失。

**输入**：一个或多个CSV文件。  
**输出**：待解析文件集合。

### Step 2：本地导入与结构识别

网页根据表头自动判断文件类型，修复BOM、引号内逗号、未转义备注和需求描述错列。不同来源映射到统一反馈模型。

**质量门**：无法识别的文件不进入分析，并明确提示文件名。  
**输出**：标准化反馈记录及参考数据。

### Step 3：NLP预处理与反馈分诊

系统规范化文本，识别噪声、情感、主题和反馈类型。输出咨询、问题/Bug、功能需求、体验问题、表扬、一般反馈或噪声。

**质量门**：高风险短文本不能因字数少被误删；星级只作为弱特征。  
**输出**：带置信度、严重度和信号强度的反馈原子。

### Step 4：共性问题聚合

相同业务本质的反馈跨平台、跨用户聚合为主题簇，计算反馈数、独立用户数、渠道数、负面率和时间趋势，并选择高信号代表原文。

**质量门**：每个Top主题至少保留3条可追溯证据。  
**输出**：共性问题簇和证据包。

### Step 5：热点和价值排序

系统综合规模、负面、严重度、影响面、趋势和客户价值生成优先分。产品经理查看分项依据，确认是否符合业务事实。

**人工检查**：金额、隐私、整批失败、VIP续约和签约前提。  
**输出**：Top问题清单。

### Step 6：行动路由

| 判断条件 | 路由 | 产出 |
|---|---|---|
| FAQ可回答的咨询 | 客服审核 | 知识库回复草稿 |
| 可复现的产品异常 | 研发/算法 | Bug行动卡片 |
| 与需求池高相似 | 产品经理 | 关联已有REQ |
| 未发现高相似需求 | 需求评审 | 候选新需求 |
| 已修复但仍有反馈 | 责任团队 | 观察复发/回归验证 |
| 噪声或无行动价值 | 归档 | 不进入行动中心 |

### Step 7：人工审核

产品经理或客服检查原文、代表证据、客户身份、知识事实和建议措辞。必要时修改主题、优先级、责任人和回复草稿。

**禁止自动通过**：财务风险、隐私安全、VIP续约、补偿、责任归属和上线时间。  
**输出**：已确认行动对象。

### Step 8：执行与状态回写

经确认的行动进入客服回复、Bug处理或需求评审。责任团队补充状态：待确认、处理中、修复验证中、已解决、已关联需求或暂缓。

**输出**：正式处理状态和责任人。

### Step 9：效果监控

下一批CSV导入后，系统比较主题规模、负面率和复发情况：

- 修复后反馈下降：验证措施有效；
- 修复后仍持续：重新打开问题并补充样本；
- 新主题快速上升：进入人工复核和新问题评估；
- 回复采纳率低：复核知识口径和生成模板。

### Step 10：反馈学习

人工修改的主题、情感、优先级和查重结果进入标注池，用于迭代词库、规则、分类器和聚类模型。最终形成“数据—AI—人工—行动—新数据”的持续学习闭环。

## 4. 状态机

| 状态 | 进入条件 | 退出条件 |
|---|---|---|
| 待分析 | CSV导入成功 | NLP处理完成 |
| 待分诊 | 已生成标签和主题 | 路由完成 |
| 待审核 | 高价值或高风险行动 | 人工确认/驳回 |
| 已关联 | 命中已有需求或Bug | 需求/缺陷状态更新 |
| 处理中 | 已分配责任团队 | 修复、回复或评审完成 |
| 验证中 | 已发布修复或临时方案 | 新数据证明有效/复发 |
| 已关闭 | 问题解决或无行动价值 | 复发时重新打开 |

## 5. 异常流程

- **CSV无法识别**：提示缺失表头，不覆盖当前看板数据；
- **一行多个事项**：标记为待拆分，人工拆分或后续由LLM拆分；
- **主题低置信度**：进入“其他体验/人工复核”，不直接建单；
- **知识库无答案**：只生成信息收集话术，不编造事实；
- **需求相似度中等**：同时展示候选REQ和原文，由产品经理决定；
- **模型与星级冲突**：展示冲突标记，优先审核文本语义；
- **已修复问题复发**：保留原问题关联，重新进入验证与回归流程。

## 6. 建议运营节奏

| 频率 | 动作 |
|---|---|
| 每日 | 导入新增评论和工单，处理高风险提醒 |
| 每周 | 产品、客服、研发共同复核Top问题与上升主题 |
| 每两周 | 清理需求查重结果，更新问题状态和临时方案 |
| 每月 | 评估主题/情感质量，复核词库和排序权重 |
| 每版本 | 比较发布前后主题规模、负面率和复发率 |
